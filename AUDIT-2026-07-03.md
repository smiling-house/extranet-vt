# Codebase Audit — VillaTracker Extranet
**Date:** 2026-07-03 · **Branch audited:** `main` (via `claude/system-status-check-5okdv4`)
**Scope:** full repo — build health, security, correctness, code quality, dependencies.

## Verdict: "Is everything working?"

**The app builds and deploys, but three live credentials ship inside the public JS bundle, the documented install path is broken, and the payment flow can charge a card and then silently lose the reservation.**

| Check | Result |
|---|---|
| Production build (`react-scripts build`) | ✅ Compiles; deployable `build/` produced |
| Stock `npm install` (incl. `Frontend install.bat`) | ❌ **Fails** — `react-dates@21` peer-conflicts with React 18; requires `--legacy-peer-deps` (or yarn) |
| Bundle size | ⚠️ 1.26 MB gzipped main chunk; no code splitting (`React.lazy` unused across ~130k lines) |
| Tests | ❌ One test exists (CRA boilerplate `App.test.js`) and **it fails** — asserts "learn react" text that isn't rendered |
| Secrets in shipped bundle | ❌ **Confirmed present in compiled `main.*.js`** (see §1) |

## 1. CRITICAL — rotate these credentials today

All three were verified present in the compiled production bundle, i.e. served to every visitor of the deployed site.

1. **Live OpenAI secret key** hardcoded in **8 files** with `dangerouslyAllowBrowser: true`
   (`src/components/AI/index.jsx:6`, `src/Util/general.js:6`, `src/pages/{Listings2,ListingsRU,ListingsBP,SearchListings,ListingsRU_DECLINED_BUT_LISTED_ON_RU,ListingsBP_DECLINED_BUT_LISTED_ON_BP}/row/listingRow.js`).
   Anyone can extract it and bill your OpenAI org. **Rotate at platform.openai.com, then proxy AI calls through the backend.**

2. **SUPER_ADMIN backend JWT, valid until ~2042**, hardcoded at `src/Util/constants.js:120` (`SHUB_TOKEN`) and inlined across ~49 files (`src/core/index.js:6`, `src/services/auth.service.js:11,146,443,498,512` — line 443 carries a *second, different* admin JWT). Payload: `ROLE_SUPER_ADMIN`, exp 2042. Every browser that loads the app holds super-admin API access to `api.villatracker.com` (partner CRUD, resync, bulk email). A code comment shows it was deliberately re-issued 2026-07-02. **Rotate/revoke server-side; per-user short-lived tokens only.**

3. **Backend `X-API-KEY`** hardcoded at `src/Util/constants.js:122` (also `auth.service.js:147`, `DuplicatedListings/index.jsx:49`, `RebuildImageHashesPopup.jsx:253`). **Rotate.**

No frontend refactor matters until these are rotated — the keys must be treated as already compromised.

## 2. HIGH — auth model & the bulk-email tool

- **Admin access is decided client-side.** Routes (incl. `/admin-settings`, `src/pages/index.jsx:522`) render with no guard; the "role" is a plain string in localStorage (`src/store/redux/User/actions.js:27`, `components/Layout/index.js:28`). The sidebar merely *hides* admin links; direct navigation works for anyone logged in.
- **Bulk welcome-email tool** (`src/pages/AdminSettings/index.jsx:41-44,194,268-284`) uses the shared hardcoded admin token to hit `local/bulk-welcome-emails/start` with `cohort: "all-with-email"` + a `force` re-send flag, and downloads a CSV of partner IDs + emails. Combined with the above: **any logged-in partner can mass-email every partner and exfiltrate the partner email list.** Fix belongs on the backend: endpoint must require a real admin identity.
- JWT + agent PII persisted in localStorage with no expiry/refresh handling.

## 3. Correctness bugs (user-visible risk)

- **Payments — `src/pages/PropertyReserve/CheckoutForm.js`:**
  - `makeReservation` (≈246–293) runs *after* `stripe.confirmCardPayment` succeeds but has no try/catch → if `/reservation/add-reservation` fails, **the customer is charged and the reservation is silently lost**.
  - `handleSubmit` (≈295–331): a failed `create-payment-intent` leaves the button stuck in "processing" forever.
  - `makeGuestyReservation` (≈180–244) posts a **hardcoded test payload** (dates `2023-05-17/19`, `reservationID: "123"`, `"string"` placeholders) to `/shub/new-reservation` and toasts success — test scaffolding live in the payment path.
- **Login — `src/store/redux/User/service.js:31-39` (and 74, 114, 135):** `catch` handlers read `response.response.data.message`; on a network error that's undefined → the catch itself throws → login button hangs with no message. `:160` swallows an error entirely.
- **App boot — `src/pages/index.jsx`:**
  - `:184` `const [profile, setProfile] = [];` — `setProfile` is `undefined` yet passed as a prop to ~15 pages (latent crash).
  - `getProfile()` (227–274) un-caught in `useEffect`; profile-fetch failure = white screen, no redirect.
  - `:247-249` writes the *stale* `agency` value to `localStorage.travelAgency` — which the **payment form** later reads.
  - `:395` non-exact `PATH_PROPERTY` route shadows the `exact` one at `:479` and any `/property/:id` routes.
  - Resize listener added without cleanup (295–299); unknown URLs render the sidebar shell instead of a 404.
- **App.test.js fails**; 39 broken relative imports exist in dead/commented code (e.g. `components/Menu/index.js:3`, `pages/Touch/NavbarTouch.jsx`) — harmless today, landmines if wired up.

## 4. Duplication — ~25k lines that are one page

The Partners cluster is **28,822 lines across 9 directories that are essentially the same page**:
- `Partners` vs `PartnersSH`: ~95% identical · vs `PartnersVT`: ~98% · `PartnersBART` vs `PartnersINVENIO`: **~99.5%** · `ListingsBP` vs `ListingsRU`: ~98%.
- Sub-trees (`ApproveAgent/`, `EditPartner/`, `BankDetails/`, …) are **byte-identical** copies (18 files / ~4k lines per channel).
- **The fix is already half-shipped:** `PartnersBartV2`, `PartnersInvenioV2`, `PartnersBookingpalV2`, `PartnersGuestyDH`, `PartnersRuDH` are 22-line wrappers around the shared `src/pages/PartnersListView`. Finishing that migration for Partners/SH/VT/BP/RU (+ the Listings variants) deletes ~25,000 lines.

## 5. Dead weight

- **1,659 `console.log` calls in 185 files** — many log tokens, agent records, and full API responses to every visitor's console.
- **~8,900 lines of dead files:** `EPartnersReservations-OLD-BACKUP/`, `Property copy/`, `Profile-old/`, `*_old`/`*BACKUP` files, `store/reducers-old/`, unused `store/redux/{agent,agency}/`, `jsondiffSH/` JSON dumps, `pages/Admin/get 9000 props.js`.
- **Unrouted legacy pages still imported** in `pages/index.jsx:26-30,63` (~14k lines compiled into the bundle for nothing).
- `src/assets` is **41 MB**.
- ~1,700 commented-out statements + 488 commented JSX blocks.

## 6. Dependencies

- **13 removable (zero imports):** `mapbox-gl`, `google-map-marker`, `react-datepicker`, `@bcad1591/react-date-picker`, `tw-daterange`, `react-multi-carousel`, `react-responsive-combo-box`, `framer-motion`, `jwt-decode`, `youtube-video-js`, `bootstrap`, `stripe` (server-side Node SDK in a frontend!), plus one of the redundant pairs below.
- **Redundant pairs in use:** `moment` + `dayjs`; `maplibre-gl` + `@react-google-maps/api`; `react-dates` + `@mui/x-date-pickers`. Pick one of each.
- `react-dates@21` is abandoned (wants React 16) and is the reason `npm install` fails — replacing it with the already-used MUI date pickers fixes install *and* removes a dead lib.
- `.env` is committed despite `.gitignore` (only a Stripe *test publishable* key — low risk, but `git rm --cached .env` + commit a `.env.example`).
- Two Google Maps keys inlined (`pages/Property/index.js:1064`, `PropertyEdit/index.js:830`) — restrict by HTTP referrer or rotate.

## 7. Architecture notes

- Redux is vestigial: 3 reducers, ~46 `useSelector` sites vs **752 `localStorage` references in 121 files**. localStorage is the real store (auth, filters, pagination, even screen size), and everything else is prop-drilled through ~45 inline routes.
- No code splitting; one 1.26 MB gzipped chunk for a ~130k-line app.
- Zero real tests.
